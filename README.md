## 🚀 Project Overview
The **Async Weather Tracker** is a Vanilla JavaScript application that fetches real-time weather data using the Fetch API. The primary focus of this project is to demonstrate:
- **Asynchronous Flow:** Utilizing `async/await` and Promises to handle API interactions.
- **Event Loop Analysis:** Tracking the execution order of Synchronous tasks, Microtasks (Promises), and Macrotasks (`setTimeout`).
- **Data Persistence:** Using `Local Storage` to maintain a search history that persists across page refreshes.
- **Error Handling:** Gracefully managing invalid city names and network issues using `try...catch` blocks.

## 🛠️ Technology Stack
- **HTML5:** Semantic structure for the weather dashboard.
- **CSS3:** Custom styling for a professional, dark-themed UI (no frameworks used).
- **Vanilla JavaScript:** DOM manipulation, event handling, and asynchronous logic.
- **API:** OpenWeatherMap API for real-time meteorological data.

## ⚙️ Key Features
- **Weather Search:** Input a city name to get temperature, humidity, and weather conditions.
- **Dynamic Search History:** Clickable history buttons that allow for quick re-fetching of weather data.
- **Live Event Loop Console:** A custom UI panel that logs the internal execution order of JavaScript functions to visualize how the Call Stack and Task Queue operate.
- **Responsive Design:** A clean, split-panel layout optimized for desktop and lab environments.

## 🧪 Learning Objectives Achieved
1. Understanding the difference between Synchronous and Asynchronous execution.
2. Handling API states (loading, success, and error).
3. Analyzing how the Call Stack, Web APIs, and Task Queue interact during a `fetch()` call.
4. Implementing client-side storage for improved user experience.
